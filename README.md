# GalChat

## 项目概述

GalChat 是一个以**Galgame对话逻辑**为灵感设计的聊天系统实验项目。

它并不追求一次性生成“最优回答”，而是将聊天过程设计为类似 Galgame 的**选项驱动式对话体验**：

- 先出现多个可选的回复方向
- 用户做出选择
- 系统根据选择生成对应风格的回复

整个过程强调 **参与感、语气控制与情绪代入**，而非纯粹的信息输出。

## 设计理念

### 1. 对话即选择（Choice-driven Conversation）

在 Galgame 中，玩家并不是直接“说一句话”，而是：

> 在多个选项中，选择自己想要的态度、立场或情绪。

GalChat 将这一机制引入聊天场景：

- 每一次回复，本质上都是一次“分支选择”
- 用户通过选择方向，决定对话的走向和氛围

系统生成的内容，始终服务于这个选择，而不是替代它。

---

### 2. 回复方向 ≠ 具体台词

GalChat 中首先出现的不是完整句子，而是**回复方向**，就像 Galgame 中的选项文本：

> 简短、概括、指向明确，但不直接暴露具体台词。

真正的回复内容，只有在用户确认选择后才会生成。

同时，该项目也内置了一套回复词典，可供选择使用。


### 首次使用配置
1.启动程序后，点击侧边栏的  设置 按钮

2.在设置页面输入您的 OpenAI API Key

3.选择模型和配置其他参数

4.点击 保存并应用

### 基础使用流程

#### 输入消息：

在底部输入框直接输入或从剪贴板粘贴内容

点击发送按钮或按回车键

#### 获取回复：

AI回复会自动显示在对话区域

回复内容会自动复制到剪贴板

### 高级功能

- 剪贴板监听：开启后，复制任何文本都会自动填充到输入框

- 在控制台可以查看完整的API交互数据

- 支持在日志查看当前状况

## 核心特性（面向体验）

- **方向先行**：回复前先确定语气与态度
- **风格可控**：用户决定对话走向，而非 AI 自由发挥
- **低侵入感**：不说教、不总结、不主动扩展话题
- **情绪一致性**：同一轮对话中保持明确的表达风格

## 适用场景

GalChat 更适合以下类型的交流：

- 偏情绪与氛围的聊天
- 熟人或半熟人对话
- 希望“像在选 Galgame 选项一样聊天”的用户

而非用于：

- 知识检索
- 专业问答
- 长篇内容生成

---

## 项目定位

GalChat 并不是一个通用型 AI 聊天助手，而是一次：

> **将 Galgame 的对话交互逻辑引入现实聊天场景的设计尝试。**

它关注的问题不是“回答是否全面”，而是：

> “在这个场景下，这个选择是否符合情绪和关系。”

## 大饼

### 目前正在推进的方向

- **提示词的修改与优化**  
  持续调整系统提示与方向约束方式，进一步降低 AI 痕迹，使回复更贴近真实对话语感。

- **界面 UI 体验优化**  
  优化聊天界面的布局、层级与交互细节，使整体体验更接近 Galgame 式的沉浸式文字互动。

- **本地方向库的持续扩充**  
  丰富不同的回复方向，为对话提供更细腻的分支选择。

### 未来计划加入的方向

- **聊天对象的创建与管理**  
  支持创建不同的聊天对象，并为其配置独立的性格、方向偏好与对话风格。

- **本地历史记录保存**  
  支持将对话过程持久化存储，用于回顾、继续对话或长期互动体验。

- **导入第三方聊天历史**  
  支持从 QQ 等聊天软件中导入历史聊天记录，用于重建上下文演进。

- **表情包与图片的识别支持**  
  为非文本内容提供基础理解能力，使对话在表达形式上更加完整。

### Coming Soooooooon

- **通过技术手段实时获取对话内容**  
  在不打断用户正常聊天行为的前提下，实时捕获对话上下文，进一步降低使用门槛，使 GalChat 更自然地融入真实聊天场景。
